12 The `Colormap` extension provides an easier way to set up a different - static -
13 color map per-layer. This means that we can set up a map of colors for each key,
14 on a per-layer basis, and whenever a layer becomes active, the color map for
15 that layer is applied. Colors are picked from a 16-color palette, provided by
16 the [LED-Palette-Theme][plugin:l-p-t] plugin. The color map is stored in
17 `EEPROM`, and can be easily changed via the [Focus][plugin:focus] plugin, which
18 also provides palette editing capabilities.

23 ## Using the extension
24 
25 To use the extension, include the header, tell it the number of layers you have,
26 register the `Focus` hooks, and it will do the rest.
27 
28 ```c++
29 #include <Kaleidoscope.h>
30 #include <Kaleidoscope-Colormap.h>
31 #include <Kaleidoscope-Focus.h>
32 
33 void setup(void) {
34  Kaleidoscope.use(&ColormapEffect, &Focus);
35 
36  Kaleidoscope.setup();
37 
38  ColormapEffect.max_layers(1);
39  Focus.addHook(FOCUS_HOOK_LEDPALETTETHEME);
40  Focus.addHook(FOCUS_HOOK_COLORMAP);
41 }
42 ```

44 ## Plugin methods
45 
46 The extension provides an `ColormapEffect` singleton object, with a single method:
47 
48 ### `.max_layers(max)`
49 
50 > Tells the extension to reserve space in EEPROM for up to `max` layers. Can
51 > only be called once, any subsequent call will be a no-op.
52 
53 ## Focus commands
54 
55 The plugin provides a single `Focus` hook, `FOCUS_HOOK_COLORMAP`, implementing
56 the following command:
57 
58 ### `colormap.map`
59 
60 > Without arguments, prints the color map: palette indexes for all layers.
61 >
62 > With arguments, updates the color map with new indexes. One does not need to
63 > give the full map, the plugin will process as many arguments as available, and
64 > ignore anything past the last key on the last layer (as set by the
65 > `.max_layers()` method).
