12 The `Unicode` extension makes it easier to write plugins that input Unicode
13 symbols on the host. Because inputting Unicode varies from OS to OS, this helper
14 library was made to hide most of the differences. All one has to do, is set up
15 the `HostOS` singleton properly, and the `Unicode` library will handle the rest,
16 by providing an easy interface for inputting Unicode symbols by their 32-bit
17 codepoints.
18 
19 ## Using the extension
20 
21 Using the extension is as simple as including the header, registering it with
22 `Kaleidoscope.use()`, and then using any of the methods provided by the
23 `Unicode` singleton object.
24 
25 ```c++
26 #include <Kaleidoscope.h>
27 #include <Kaleidoscope-Unicode.h>
28 
29 void setup() {
30  Kaleidoscope.use(&Unicode);
31  Kaleidoscope.setup();
32 
33  Unicode.type(0x2328);
34 }
35 ```

37 ## Extension methods
38 
39 The extension provides a number of methods on the `Unicode` object, but also has
40 symbols that can be [overridden](#overrideable-methods), to add custom
41 functionality.
42 
43 ### `.type(code_point)`
44 
45 > Starts the Unicode input method using the [`.start()`](#start) method, inputs
46 > the `code_point` using [`.typeCode()`](#typeCode), and finishes up with
47 > the [`.end()`](#end) method. For each hexadecimal digit sent to the host,
48 > the [`.input()`](#input) method will also be called.
49 >
50 > This method is most useful when one knows the code point of the Unicode symbol
51 > to enter ahead of time, when the code point does not depend on anything else.
52 
53 ### `.typeCode(code_point)`
54 
55 > Inputs the hex codes for `code_point`, and the hex codes only. Use when the
56 > input method is to be started and ended separately.
57 >
58 > For example, a macro that starts Unicode input, and switches to a layer full
59 > of macros that send the hex codes is one scenario where this function is of
60 > use.
61 
62 ### `.start()`
63 
64 > Starts the Unicode input method. The way it starts it, depends on the host
65 > operating system.
66 
67 ### `.input()`
68 
69 > If the host operating system requires keys being held during the Unicode
70 > input, this function will hold them for us.
71 
72 ### `.end()`
73 
74 > Finishes the Unicode input method, in an OS-specific way.
75 
76 ## Overrideable methods
77 
78 ### `hexToKey(hex_digit)`
79 
80 > A function that returns a `Key` struct, given a 8-bit hex digit. For most
81 > uses, the built-in version of this function is sufficient, but if the keymap
82 > on the OS-side has any of the hexadecimal symbols on other scancodes than
83 > QWERTY, this function should be overridden to use the correct scan codes.
84 
85 ### `unicodeCustomStart()`
86 
87 > If the host OS type is set to `kaleidoscope::hostos::Custom`, then this function will
88 > be called whenever the [`.start()`](#start) method is called. The default
89 > implementation does nothing, and should be overridden to implement the custom
90 > magic needed to enter unicode input mode.
91 
92 ### `unicodeCustomInput()`
93 
94 > If the host OS type is set to `kaleidoscope::hostos::Custom`, then this function will
95 > be called whenever the [`.input()`](#input) method is called. The default
96 > implementation does nothing, and should be overridden to implement the custom
97 > magic needed while inputting the hex code itself (such as holding additional
98 > keys).
99 
100 ### `unicodeCustomEnd()`
101 
102 > If the host OS type is set to `kaleidoscope::hostos::Custom`, then this function will
103 > be called whenever the [`.end()`](#end) method is called. The default
104 > implementation does nothing, and should be overridden to implement the custom
105 > magic needed to leave unicode input mode.
