12 Syster is a way to input symbols in a different way: instead of macros, Leader
13 sequences or the like, we trigger the special input mode, and enter the symbol's
14 name. Once finished, we hit `Space`, and this plugin will do the rest: delete
15 everything we typed, look up an action for the entered symbol, and execute that.
16 
17 There are a number of ways this can be useful, but the easiest showcase is
18 symbolic Unicode input: `SYSTER coffee SPACE` turns into `☕`, with just a
19 little code.
20 
21 ## Using the plugin
22 
23 To use the plugin, one needs to include the header and set up a function that
24 will handle the symbol actions:
25 
26 ```c++
27 #include <Kaleidoscope.h>
28 #include <Kaleidoscope-Syster.h>
29 #include <Kaleidoscope-Unicode.h>
30 #include <kaleidoscope/hid.h>
31 
32 void systerAction(kaleidoscope::Syster::action_t action, const char *symbol) {
33  switch (action) {
34  case kaleidoscope::Syster::StartAction:
35  Unicode.type (0x2328);
36  break;
37  case kaleidoscope::Syster::EndAction:
38  handleKeyswitchEvent (Key_Backspace, UNKNOWN_KEYSWITCH_LOCATION, IS_PRESSED | INJECTED);
39  kaleidoscope::hid::sendKeyboardReport ();
40  handleKeyswitchEvent (Key_Backspace, UNKNOWN_KEYSWITCH_LOCATION, WAS_PRESSED | INJECTED);
41  kaleidoscope::hid::sendKeyboardReport ();
42  break;
43  case kaleidoscope::Syster::SymbolAction:
44  Serial.print ("systerAction: symbol=");
45  Serial.println (symbol);
46  if (strcmp (symbol, "coffee") == 0) {
47  Unicode.type (0x2615);
48  }
49  break;
50  }
51 }
52 
53 void setup() {
54  Serial.begin(9600);
55 
56  Kaleidoscope.use(&Unicode, &Syster);
57 
58  Kaleidoscope.setup ();
59 }
60 ```
61 
62 **Note** that we need to use the `Syster` object before any other that adds or
63 changes key behaviour! Failing to do so may result in unpredictable behaviour.

65 ## Plugin methods
66 
67 The plugin provides the `Syster` object, with no public methods. There are two
68 methods outside of the object, however, that can be overridden:
69 
70 ### `systerAction(action, symbol)`
71 
72 > Called whenever an action needs to be taken, which can happen in three cases:
73 
74 > First, when the `Syster` key is pressed and the alternate processing starts.
75 > In this case, `action` will be set to `kaleidoscope::Syster::StartAction`, and
76 > `symbol` will be `NULL`. This function can be used to do some setup to make it
77 > more obvious that the Syster input mode is active, such as sending a Unicode
78 > symbol to the host, or lighting up LEDs, or anything else we'd like.
79 >
80 > Second, when the sequence is finished with a `Space`. In this case, `action`
81 > will be set to `kaleidoscope::Syster::EndAction` and `symbol` will be `NULL`.
82 > This can be used to undo anything that the start action did, if need be.
83 >
84 > Third, when the action for the symbol should be made. In this case, `action`
85 > is set to `kaleidoscope::Syster::SymbolAction`, and `symbol` will be a C
86 > string. It is up to us, what we do with this information, how we handle it.
87 
88 ### `keyToChar(key)`
89 
90 > A function that turns a keycode into a character. If using QWERTY on the host,
91 > the default implementation is sufficient. When using something else, you may
92 > have to reimplement this function.
