12 The `MagicCombo` extension provides a way to perform custom actions when a
13 particular set of keys are held down together. The functionality assigned to
14 these keys are not changed, and the custom action triggers as long as all keys
15 within the set are pressed. The order in which they were pressed do not matter.
16 
17 This can be used to tie complex actions to key chords.
18 
19 ## Using the extension
20 
21 To use the extension, we must include the header, create an array of combos we
22 want to work with, let the plugin know we want to work with those, and then use
23 a special function to handle the combos:
24 
25 ```c++
26 #include <Kaleidoscope.h>
27 #include <Kaleidoscope-MagicCombo.h>
28 
29 static const KaleidoscopePlugins::MagicCombo::combo_t magic_combos[] PROGMEM = {
30  {R1C3 | R2C1 | R2C4 | R2C7, // left hand,
31  R0C11 | R1C12 | R2C14 //right hand
32  },
33  {0, 0}
34 };
35 
36 void magicComboActions(uint8_t combo_index, uint32_t left_hand, uint32_t right_hand) {
37  switch (combo_index) {
38  case 0:
39  Serial.println("It's a kind of magic!");
40  break;
41  }
42 }
43 
44 void setup() {
45  Serial.begin(9600);
46 
47  Kaleidoscope.use(&MagicCombo);
48 
49  Kaleidoscope.setup();
50 
51  MagicCombo.magic_combos = magic_combos;
52 }
53 ```
54 
55 The combo list **must** reside in `PROGMEM`, and is a list of tuples. Each
56 element in the array has two fields: the left hand state, and the right hand
57 state upon which to trigger the custom action. Both of these are bit fields,
58 each bit set tells the extension that the key with that index must be held for
59 the action to trigger. It is recommended to use the `RxCy` macros of the core
60 `KaleidoscopeFirmware`, and *or* them together to form a bitfield.
61 
62 The combo list **must** end with an element containing zero values for both the
63 left and the right halves.

65 ## Extension methods
66 
67 The extension provides a `MagicCombo` singleton object, with the following
68 methods and properties:
69 
70 ### `.magic_combos`
71 
72 > Setting this property lets the plugin know which combinations of key presses
73 > we are interested in. If any of these are found active, the
74 > `magicComboActions()` function will be called.
75 
76 ### `.min_interval`
77 
78 > Restrict the magic action to fire at most once every `minInterval`
79 > milliseconds.
80 >
81 > Defaults to 500.
82 
83 ## Overrideable methods
84 
85 Whenever an combination is found to be held, the extension will trigger an
86 action, in each scan cycle until the keys remain held. This is done by calling
87 the overrideable `magicComboActions` function:
88 
89 ### `magicComboActions(combo_index, left_hand, right_hand)`
90 
91 > Called whenever a combination is found to be held. The function by default
92 > does nothing, and it is recommended to override it from within the Sketch.
93 >
94 > The first argument will be the index in the combo list, the other two are the
95 > key states on the left and right halves, respectively.
96 >
97 > Plugins that build upon this extensions *should not* override this function,
98 > but provide helpers that can be called from it. An override should only happen
99 > in the Sketch.
