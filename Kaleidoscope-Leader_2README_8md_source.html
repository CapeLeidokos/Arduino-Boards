12 Leader keys are a kind of key where when they are tapped, all following keys are
13 swallowed, until the plugin finds a matching sequence in the dictionary, it
14 times out, or fails to find any possibilities. When a sequence is found, the
15 corresponding action is executed, but the processing still continues. If any key
16 is pressed that is not the continuation of the existing sequence, processing
17 aborts, and the key is handled normally.
18 
19 This behaviour is best described with an example. Suppose we want a behaviour
20 where `LEAD u` starts unicode input mode, and `LEAD u h e a r t` should result
21 in a heart symbol being input, and we want `LEAD u 0 0 e 9 SPC` to input `é`,
22 and any other hex code that follows `LEAD u`, should be handled as-is, and
23 passed to the host. Obviously, we can't have all of this in a dictionary.
24 
25 So we put `LEAD u` and `LEAD u h e a r t` in the dictionary only. The first will
26 start unicode input mode, the second will type in the magic sequence that
27 results in the symbol, and then aborts the leader sequence processing. With this
28 setup, if we type `LEAD u 0`, then `LEAD u` will be handled first, and start
29 unicode input mode. Then, at the `0`, the plugin notices it is not part of any
30 sequence, so aborts leader processing, and passes the key on as-is, and it ends
31 up being sent to the host. Thus, we covered all the cases of our scenario!

33 ## Using the plugin
34 
35 To use the plugin, one needs to include the header, implement some actions,
36 create a dictionary, and configure the provided `Leader` object to use the
37 dictionary:
38 
39 ```c++
40 #include <Kaleidoscope.h>
41 #include <Kaleidoscope-Leader.h>
42 
43 static void leaderA() {
44  Serial.println("leaderA");
45 }
46 
47 static void leaderTX() {
48  Serial.println("leaderTX");
49 }
50 
51 static const kaleidoscope::Leader::dictionary_t leader_dictionary PROGMEM =
52  LEADER_DICT({LEADER_SEQ(LEAD(0), Key_A), leaderA},
53  {LEADER_SEQ(LEAD(0), Key_T, Key_X), leaderTX});
54 
55 void setup() {
56  Serial.begin(9600);
57 
58  Kaleidoscope.use(&Leader);
59 
60  Kaleidoscope.setup();
61 
62  Leader.dictionary = leader_dictionary;
63 }
64 ```
65 
66 The dictionary is made up of a list of keys, and an action callback. Using the
67 `LEADER_DICT` and `LEADER_SEQ` helpers is recommended. The dictionary *must* be
68 marked `PROGMEM`!
69 
70 **Note** that we need to use the `Leader` object before any other that adds or
71 changes key behaviour! Failing to do so may result in unpredictable behaviour.

73 ## Plugin methods
74 
75 The plugin provides the `Leader` object, with the following methods and properties:
76 
77 ### `.dictionary`
78 
79 > Set this property to the dictionary `Leader` should use. The dictionary is an
80 > array of `kaleidoscope::Leader::dictionary_t` elements. Each element is made
81 > up of two elements, the first being a list of keys, the second an action to
82 > perform when the sequence is found.i
83 >
84 > The dictionary *MUST* reside in `PROGMEM`.
85 
86 ### `.reset()`
87 
88 > Finishes the leader sequence processing. This is best called from actions that
89 > are final actions, where one does not wish to continue the leader sequence
90 > further in the hopes of finding a longer match.
91 
92 ### `.time_out`
93 
94 > The number of milliseconds to wait before a sequence times out. Once the
95 > sequence timed out, if there is a partial match with an action, that will be
96 > performed, otherwise the Leader sequence will simply reset.
97 >
98 > Defaults to 1000.
