12 The `HostOS` extension is not all that useful in itself, rather, it is a
13 building block other plugins and extensions can use to not repeat the same
14 guesswork and logic. Its primary purpose is to help either detect, or keep track
15 of the host operating system. The detection part is not the most reliable thing,
16 mind you.
17 
18 The goal is to have a single place that remembers the host OS, either detected,
19 or set by the end-user, in a Sketch, or via a macro, or some other way. This
20 information can then be reused by other plugins.
21 
22 See the [Unicode][plugin:unicode] extension for an example about how to use
23 `HostOS` in practice.
24 
25  [plugin:unicode]: https://github.com/keyboardio/Kaleidoscope-Unicode

27 ## Using the extension
28 
29 The extension provides a `HostOS` singleton object. It can either be a simple
30 one without auto-detection (the default), or one that will try to detect the
31 Host OS, using the [FingerprintUSBHost][fprdetect] library. To enable
32 auto-detection, `KALEIDOSCOPE_HOSTOS_GUESSER` must be defined before including
33 the `HostOS` library header.
34 
35  [fprdetect]: https://github.com/keyboardio/FingerprintUSBHost
36 
37 ```c++
38 #define KALEIDOSCOPE_HOSTOS_GUESSER 1
39 
40 #include <Kaleidoscope.h>
41 #include <Kaleidoscope-HostOS.h>
42 #include <Kaleidoscope/HostOS-select.h>
43 
44 void someFunction(void) {
45  if (HostOS.os() == kaleidoscope::HostOS::LINUX) {
46  // do something linux-y
47  }
48  if (HostOS.os() == kaleidoscope::HostOS::OSX) {
49  // do something OSX-y
50  }
51 }
52 
53 void setup(void) {
54  Kaleidoscope.use(&HostOS);
55 
56  Kaleidoscope.setup ();
57 }
58 ```
59 
60 To be able to choose between the two variants, one must also include the
61 `Kaleidoscope/HostOS-select.h` header.

63 ## Extension methods
64 
65 The extension provides the following methods on the `HostOS` singleton:
66 
67 ### `.os()`
68 
69 > Returns the stored type of the Host OS.
70 
71 ### `.os(type)`
72 
73 > Sets the type of the host OS, overriding any previous value. The type is then
74 > stored in EEPROM for persistence.
