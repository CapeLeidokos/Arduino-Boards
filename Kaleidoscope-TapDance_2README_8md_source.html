12 Tap-dance keys are general purpose, multi-use keys, which trigger a different
13 action based on the number of times they were tapped in sequence. As an example
14 to make this clearer, one can have a key that inputs `A` when tapped once,
15 inputs `B` when tapped twice, and lights up the keyboard in Christmas colors
16 when tapped a third time.
17 
18 This behaviour is most useful in cases where we have a number of things we
19 perform rarely, where tapping a single key repeatedly is not counter-productive.
20 Such cases include - for example - multimedia forward / backward keys: forward
21 on single tap, backward on double. Of course, one could use modifiers to achieve
22 a similar effect, but that's two keys to use, this is only one. We can also hide
23 some destructive functionality behind a number of taps: reset the keyboard after
24 4 taps, and light up LEDs in increasingly frightful colors until then.
25 
26 ## How does it work?
27 
28 To not interfere with normal typing, tap-dance keys have two ways to decide when
29 to call an action: they either get interrupted, or they time out. Every time a
30 tap-dance key is pressed, the timer resets, so one does not have to finish the
31 whole tapping sequence within a short time limit. The tap-dance counter
32 continues incrementing until one of these cases happen.
33 
34 When a tap-dance key is pressed and released, and nothing is pressed on the
35 keyboard until the timeout is reached, then the key will time out, and trigger
36 an action. Which action, depends on the number of times it has been tapped up
37 until this point.
38 
39 When a tap-dance key is pressed and released, and another key is hit before the
40 timer expires, then the tap-dance key will trigger an action first, perform it,
41 and only then will the firmware continue handling the interrupting key press.
42 This is to preserve the order of keys pressed.
43 
44 In both of these cases, the [`tapDanceAction`][tdaction] will be called, with
45 `tapDanceIndex` set to the index of the tap-dance action (as set in the keymap),
46 the `tapCount`, and `tapDanceAction` set to either
47 `kaleidoscope::TapDance::Interrupt`, or `kaleidoscope::TapDance::Timeout`. If we
48 continue holding the key, then as long as it is held, the same function will be
49 called with `tapDanceAction` set to `kaleidoscope::TapDance::Hold`. When the key
50 is released, after either an `Interrupt` or `Timeout` action was triggered, the
51 function will be called with `tapDanceAction` set to
52 `kaleidoscope::TapDance::Release`.
53 
54 These actions allow us to create sophisticated tap-dance setups, where one can
55 tap a key twice and hold it, and have it repeat, for example.
56 
57 There is one additional value the `tapDanceAction` parameter can take:
58 `kaleidoscope::TapDance::Tap`. It is called with this argument for each and
59 every tap, even if no action is to be triggered yet. This is so that we can have
60 a way to do some side-effects, like light up LEDs to show progress, and so on.

62 ## Using the plugin
63 
64 To use the plugin, we need to include the header, and declare the behaviour
65 used. Then, we need to place tap-dance keys on the keymap. And finally, we need
66 to implement the [`tapDanceAction`][tdaction] function that gets called each
67 time an action is to be performed.
68 
69 ```c++
70 #include <Kaleidoscope.h>
71 #include <Kaleidoscope-TapDance.h>
72 
73 // Somewhere in the keymap:
74 TD(0)
75 
76 // later in the Sketch:
77 void tapDanceAction(uint8_t tap_dance_index, byte row, byte col, uint8_t tap_count,
78  kaleidoscope::TapDance::ActionType tap_dance_action) {
79  switch (tap_dance_index) {
80  case 0:
81  return tapDanceActionKeys(tap_count, tap_dance_action,
82  Consumer_ScanNextTrack, Consumer_ScanPreviousTrack);
83  }
84 }
85 
86 void setup() {
87  Kaleidoscope.use(&TapDance);
88 
89  Kaleidoscope.setup ();
90 }
91 ```

106 ## Plugin methods
107 
108 The plugin provides a `TapDance` object, but to implement the actions, we need
109 to define a function ([`tapDanceAction`][tdaction]) outside of the object. A
110 handler, of sorts. Nevertheless, the plugin provides one macro that is
111 particularly useful: `tapDanceActionKeys`. Apart from that, it provides one
112 property only:
113 
114 ### `.time_out`
115 
116 > The number of loop iterations to wait before a tap-dance sequence times out.
117 > Once the sequence timed out, the action for it will trigger, even without an
118 > interruptor. Defaults to 5, and the timer resets with every tap of the same
119 
120 ### `tapDanceActionKeys(tap_count, tap_dance_action, keys...)`
121 
122 > Sets up an action where for each subsequent tap, a different key will be
123 > chosen from the list of keys supplied in the `keys...` argument.
124 >
125 > If we have `Key_A` and `Key_B` in the list, then, if tapped once, this
126 > function will input `A`, but when tapped twice, will input `B`.
127 >
128 > When all our actions are just different keys, this is a very handy macro to
129 > use.
130 >
131 > The `tap_count` and `tap_dance_actions` parameters should be the same as the
132 > similarly named parameters of the `tapDanceAction` function.
133 
134 ### `tapDanceAction(tap_dance_index, row, col, tap_count, tap_dance_action)`
135 
136 > The heart of the tap-dance plugin is the handler method. This is called every
137 > time any kind of tap-dance action is to be performed. See the
138 > *[How does it work?](#how-does-it-work)* section for details about when and
139 > how this function is called.
140 >
141 > The `tap_dance_index` and `tap_count` parameters help us choose which action
142 > to perform. The `row` and `col` parameters tell us where the tap-dance key is
143 > on the keyboard.
