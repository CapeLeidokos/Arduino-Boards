12 A common base for plugins that want to provide themes, or theme-related
13 capabilities, using a 16 color palette. In other words, this is for plugin
14 authors primarily. The primary aim of the plugin is to provide not only a common
15 palette, but tools that make it easier to use it too.

17 ## Using the plugin
18 
19 To use the plugin, one needs to do a bit more than include the header, and tell
20 the firmware to use it. Itself being a mere building block, to use it to its
21 full extent, we need to create our own plugin on top of it.

82 ## Plugin methods
83 
84 The plugin provides the `LEDPaletteTheme` object, which has the following methods and properties:
85 
86 ### `.reserveThemes(max_themes)`
87 
88 > Reserve space in EEPROM for `max_themes`. Each key on a theme uses half a byte
89 > of space. The function returns the `theme_base` to be used with the rest of
90 > the methods.
91 >
92 > The `theme_base` is a pointer into the EEPROM where the theme storage starts.
93 
94 ### `.updateHandler(theme_base, theme)`
95 
96 > A helper we can call in our plugin's `.update()` method: given an EEPROM
97 > location (`theme_base`), and a `theme` index, it will update the keyboard with
98 > the colors of the specified theme.
99 >
100 > The `theme` argument can be any index between zero and `max_themes`. How the
101 > plugin decides which theme to display depends entirely on the plugin.
102 
103 ### `.themeFocusHandler(command, expected_command, theme_base, max_themes)`
104 
105 > To be used in a custom `Focus` handler: handles the `expected_command` Focus
106 > command, and provides a way to query and update the themes supported by the
107 > plugin.
108 >
109 > When queried, it will list the color indexes. When used as a setter, it
110 > expects one index per key.
111 >
112 > The palette can be set via the `palette` focus command, implemented by the
113 > `FOCUS_HOOK_LEDPALETTETHEME` hook explained below.
114 
115 ## Focus commands
116 
117 The plugin provides a single `Focus` hook, `FOCUS_HOOK_LEDPALETTETHEME`,
118 implementing the following command:
119 
120 ### `palette`
121 
122 > Without arguments, prints the palette: RGB values for all 16 colors.
123 >
124 > With arguments, updates the palette with new colors. One does not need to give
125 > the full palette, the plugin will process as many arguments as available, and
126 > ignore anything past the last index. It expects colors to have all three
127 > components specified, or none at all. Thus, partial palette updates are
128 > possible, but only on the color level, not at component level.
